# 요구사항 분석

## 1. 프로젝트 개요

### 1.1 서비스 정의
- **서비스명**: 하루인지 (Ai-TIVE WELLNESS)
- **컨셉**: 정서자극 기반 하루 인지루틴
- **목적**: 어르신의 치매 및 노인성 질환 예방을 위한 7CORE 통합 인지회복 프로그램
- **현황**: 현재 종이(지류) 형태로 매주 배포되어 실제 운영 중인 검증된 프로그램
- **목표**: 검증된 종이 프로그램을 온라인(디지털) 플랫폼으로 전환·배포

### 1.2 타겟 사용자
- **1차 사용자**: 고령자(어르신) — 직접 프로그램을 수행
- **2차 사용자(추정)**: 보호자/가족, 요양시설 종사자, 복지센터 운영자 — 사용 보조 및 진행 점검
- **운영자**: 콘텐츠 제작·배포 담당자 (주간 발행 체계)

## 2. 콘텐츠 구조 분석

### 2.1 7일 인지루틴 (7CORE)

| Day | 영역 | 활동 방법 | 자극 영역 | 인지 효과 |
|-----|------|-----------|-----------|-----------|
| 1 | 생활체조 / 마음체조 | 음악과 함께하는 신체 활동 | 신체활동 → 뇌자극 | 전두엽, 운동피질 |
| 2 | 오감 테라피 | 다양한 오감 자극 | 감각 → 기억 활성화 | 대뇌변연계, 해마 |
| 3 | 인지 테라피 | 사고·문제 풀이 | 집중 → 인지 자극 | 전두엽 |
| 4 | 소리 테라피 | 회상중심 스토리 뮤직 | 소리 → 정서 안정 | 청각피질, 자율신경, 우뇌 |
| 5 | 아트 테라피 | 명화 미술 활동 | 감성·색채 → 감정 표현 | 우뇌, 공간 지각 |
| 6 | 문학 테라피 | 문학 읽기 / 쓰기 | 공감각 인지 자극 | 측두엽, 언어중추 |
| 7 | 일기쓰기 | 하루 체크·단기 기억 | 자기 인식 → 자기존중감 | 기억 통합 네트워크 |

### 2.2 단위 콘텐츠(페이지) 공통 구조
- **헤더**: 카테고리 태그(건강체조/마음체조/오감/인지/소리/아트/문학읽기/문학쓰기/일기), 일련번호, 작성 메타(년·월·일·요일·성함), 가이드 영상 QR
- **본문**: 활동 안내문(말벗하기), 이미지/명화/만다라/한자/사진, 인터랙션 영역(빈칸·색칠·체크박스·자유글)
- **푸터**: 정답 노출(현재 지류는 역방향 인쇄)

### 2.3 활동(인터랙션) 유형
1. **영상 시청형**: 가이드 영상을 보며 따라하기 (체조, 명상)
2. **빈칸 채우기**: 속담·단어·연도 등 텍스트 입력 후 정답 확인
3. **시각 식별형**: 같은 모양의 만다라 찾기 등 선택형
4. **색칠하기**: 만다라/명화/캐릭터를 자유롭게 채색
5. **따라쓰기**: 애국가, 명심보감, 한자(고사성어) 필사
6. **끝말잇기**: 단어 체인 입력
7. **자유 글쓰기**: 일기, 감정 단어, 회상 글
8. **체크리스트형 입력**: 감정 상태, 식사, 약 복용, 운동, 만남, 통화 기록

### 2.4 사용 흐름 (지류 기준)
1. 하루 중 일정 시간을 정해 페이지를 펼침
2. 페이지 우측 상단 QR 스캔 → 가이드 영상 시청 + 데일리 케어 진행
3. 4개의 생활 주제 완료 후 체크리스트로 인지활성화 정도 자가 점검

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 사용자(어르신) 기능
- **FR-U-01** 호(주차) / 일자 단위 콘텐츠 열람
- **FR-U-02** 가이드 영상 인앱 재생 (QR 없이도 영상 도달 가능)
- **FR-U-03** 빈칸 입력 후 정답 확인 (즉시 채점·해설)
- **FR-U-04** 만다라/명화 디지털 색칠 (팔레트, 굵기, 되돌리기, 저장)
- **FR-U-05** 따라쓰기 (가이드 라인 위에 손글씨/타이핑)
- **FR-U-06** 일기 작성 (감정 선택, 식사·약·운동·만남 체크/입력, 자유글)
- **FR-U-07** 진행 현황 시각화 (오늘/이번 주 체크리스트, 7CORE 완료 배지)
- **FR-U-08** 작성 결과물 보관·재열람 (나의 색칠, 나의 일기 아카이브)
- **FR-U-09** 회상 자료 다시보기 (그때 그 시절 등 사진/영상 보관)

### 3.2 운영자(콘텐츠 관리) 기능
- **FR-A-01** 호(주차) 단위 콘텐츠 등록 (현재 종이 발행 주기 = 주 1회)
- **FR-A-02** Day(1~7) × 카테고리별 페이지 등록·수정
- **FR-A-03** 활동 유형 템플릿 선택 (빈칸/색칠/따라쓰기/체크리스트/자유글)
- **FR-A-04** 이미지·영상·QR 자산 업로드 및 매핑
- **FR-A-05** 정답·해설 등록
- **FR-A-06** 발행/예약 발행, 공개 범위(전체/특정 시설) 제어
- **FR-A-07** 사용 통계 (회원별/시설별 완료율, 활동별 참여율)

### 3.3 보호자/시설 관리자 기능 (선택, 2차)
- **FR-G-01** 어르신 진행 현황 모니터링
- **FR-G-02** 결과물(일기·색칠) 공유 받기
- **FR-G-03** 알림 (오늘의 루틴 미완료 등)

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1 사용성 (Usability) — 최우선
- **NFR-01** 어르신 친화 UI: 최소 18~20pt 본문, 큰 버튼(터치 영역 ≥ 48×48px), 고대비 색상
- **NFR-02** 단순 네비게이션: 홈 → 오늘의 루틴 → 활동 시작 (3 depth 이내)
- **NFR-03** 음성 안내(TTS) 옵션: 텍스트 읽어주기, 활동 시작 음성 가이드
- **NFR-04** 글씨 크기 조절(최소 2단계), 다크모드는 후순위
- **NFR-05** 손가락 입력을 우선 가정 (펜·마우스는 부가)

### 4.2 접근성 (Accessibility)
- **NFR-06** WCAG 2.1 AA 준수 (특히 명도 대비, 포커스 표시, 키보드 접근)
- **NFR-07** 색약 고려 색상 팔레트
- **NFR-08** 영상 자막 제공

### 4.3 성능
- **NFR-09** 페이지 첫 로딩 3초 이내 (저사양 단말 기준)
- **NFR-10** 오프라인 대응 (한 번 받은 호는 네트워크 없이 열람 가능 — PWA 캐시)

### 4.4 보안/개인정보
- **NFR-11** 일기 등 개인 기록 암호화 저장
- **NFR-12** 고령자 회원가입 간소화 (전화번호 + 인증, 또는 시설 발급 코드)
- **NFR-13** 개인정보 최소 수집 (이름, 생년월일 정도)

### 4.5 운영
- **NFR-14** 주 1회 신규 호 발행을 무중단으로 처리
- **NFR-15** 콘텐츠 백오피스 — 비개발자(콘텐츠 기획자) 단독 운영 가능

## 5. 플랫폼 선정 분석 (웹 vs 앱)

### 5.1 각 활동 유형별 기술 검토

| 활동 | 웹 가능 여부 | 비고 |
|------|-------------|------|
| 영상 재생 | ✅ 가능 | HTML5 video |
| 음악 재생 | ✅ 가능 | HTML5 audio |
| 색칠하기 | ✅ 가능 | Canvas + Pointer Events |
| 따라쓰기(손글씨) | ✅ 가능 | Canvas + Pointer Events |
| 빈칸/체크/자유글 | ✅ 가능 | 일반 폼 |
| 카메라(QR 스캔) | ⚠️ 제한적 | HTTPS + getUserMedia 필요, iOS Safari 일부 제약 |
| 푸시 알림 | ⚠️ 제한적 | iOS 16.4+ PWA만, 안정성 낮음 |
| 홈 화면 진입(아이콘) | ⚠️ 사용자가 직접 추가 필요 | 어르신에게는 부담 |

### 5.2 결론 권장안: **모바일 우선 반응형 웹(PWA) → 추후 하이브리드 앱 래퍼**

**근거**:
1. **콘텐츠 90% 이상이 웹 기술로 충분히 구현 가능** — 색칠/따라쓰기/영상/입력은 모두 웹 표준 기능
2. **주 1회 콘텐츠 업데이트** 특성상, 앱스토어 심사 주기와 충돌하지 않는 웹 배포가 유리
3. **어르신은 앱 설치·업데이트 자체가 진입장벽** — URL 한 번 전달 후 홈 화면 추가 안내가 더 단순
4. **시설/복지센터 보급 시나리오** — 태블릿·공용 PC에서도 동일 접근 가능
5. **푸시 알림·홈 아이콘 등 앱 강점이 필요해지는 시점**에 Capacitor/WebView 래퍼로 앱화 가능 (재개발 불필요)

**의사결정 보류 항목 (PoC로 확인 필요)**:
- iOS Safari에서 Canvas 색칠 성능 (저가 태블릿 기준)
- PWA 오프라인 캐시로 영상까지 담는 것이 현실적인지(용량)
- QR 스캔이 필수인지(이미 앱 내에서 페이지에 도달했다면 QR 불필요)

## 6. 데이터 모델 초안

```
호(Issue)
 ├─ 호 번호 (예: 5호)
 ├─ 주차 (예: 4주차)
 ├─ 테마 (예: 행복한 생활)
 ├─ 발행일 / 공개여부
 └─ Day[1..7]
     └─ Page[1..N]
         ├─ 카테고리 (체조/오감/인지/소리/아트/문학/일기)
         ├─ 번호 (1_1.1 형식)
         ├─ 제목
         ├─ 활동 유형 (video / fill-blank / coloring / tracing / checklist / freewrite / match)
         ├─ 가이드 영상 URL
         ├─ 본문 자산 (이미지·텍스트)
         ├─ 정답·해설
         └─ 입력 스키마 (활동 유형별)

회원(User)
 ├─ 기본정보 (이름, 생년월일, 연락처)
 ├─ 소속 (개인 / 시설명)
 └─ 진행기록(Progress)
     ├─ 호·Day·Page 단위 완료 여부
     ├─ 작성/입력 결과 (JSON)
     ├─ 색칠/필사 결과 (이미지)
     └─ 완료 시각
```

## 7. 단계별 범위 제안 (Phasing)

### Phase 1 — MVP (최소 검증)
- 호 1개(현재 5호) 전체 7일 콘텐츠 디지털 구현
- 활동 유형 4종: 영상 시청 / 빈칸 채우기 / 색칠 / 자유 일기
- 회원가입·로그인 (시설 코드 방식)
- 진행률 체크리스트
- 운영자 백오피스는 **수동 등록 가능한 최소 형태** (직접 JSON 또는 단순 폼)

### Phase 2 — 운영화
- 백오피스 정식 구축 (호·페이지 등록, 자산 업로드, 발행 예약)
- 따라쓰기·끝말잇기·매칭 등 활동 유형 추가
- 작성 결과 아카이브, 회상 자료 다시보기

### Phase 3 — 확장
- 보호자/시설 관리자 대시보드
- 사용 통계, 인지활성화 지수 리포트
- 하이브리드 앱 패키징, 푸시 알림

## 8. 리스크 및 추가 확인 필요 사항

1. **저작권**: 명화(신윤복 단오풍정 등), 88올림픽 사진, 음악(애국가, 국민건강체조), 만다라 도안의 디지털 재배포 권한
2. **개인정보 처리방침**: 고령자 본인 동의 절차, 시설 일괄 가입 시 위임 동의 처리
3. **QR의 역할 재정의**: 디지털 전환 시 QR은 불필요할 수 있음 — 종이 병행 운영 여부에 따라 결정
4. **체크리스트→인지활성화 지수**: 현재 종이에서는 정성 평가 — 디지털화하면서 수치화 알고리즘 정의 필요
5. **결과물 보관 기간**: 일기/색칠 등 사용자 자산의 보관·삭제 정책
6. **단말 보급**: 어르신이 자기 단말로 사용할지, 시설 공용 태블릿일지 — UX 분기
