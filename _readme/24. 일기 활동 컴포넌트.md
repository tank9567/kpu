# 24. 일기 활동 컴포넌트 (DiaryActivity)

> 4종 활동 중 네 번째 — 하루 체크리스트·자유 기록을 통한 일기 작성 활동.

## 개요

| 항목 | 값 |
|------|-----|
| 활동 타입 | `diary` |
| 대표 페이지 | `1_7.1 오늘 하루 어땠나요?` |
| 메인 인터랙션 | 다양한 유형의 입력 폼 (radio / checkbox / text / textarea) |
| 자동 저장 | localStorage (400ms 디바운스) |

---

## 활동 데이터 스키마

```ts
type DiaryActivity = {
  type: 'diary';
  sections: DiarySection[];
};

type DiarySection =
  | { kind: 'radio';    key: string; label: string; options: string[] }
  | { kind: 'checkbox'; key: string; label: string; options: string[] }
  | { kind: 'text';     key: string; label: string; placeholder?: string }
  | { kind: 'textarea'; key: string; label: string; placeholder?: string; rows?: number };
```

샘플 (`app/src/data/issues/5-4.json` Day 7) — 일부 발췌:
```json
{
  "type": "diary",
  "sections": [
    { "kind": "radio",    "key": "mood",     "label": "오늘 하루 기분이 어땠나요?", "options": ["아주 좋음", "괜찮음", "보통", "조금 힘듦"] },
    { "kind": "text",     "key": "moodWord", "label": "오늘의 감정 한 단어" },
    { "kind": "text",     "key": "breakfast","label": "아침" },
    { "kind": "checkbox", "key": "exercise", "label": "오늘 어떤 운동을 하셨나요?", "options": ["걷기", "체조", "스트레칭", "하지 못함"] },
    { "kind": "textarea", "key": "messageToSelf", "label": "오늘 나에게 해주고 싶은 말은?", "rows": 4 }
  ]
}
```

원본 종이 콘텐츠의 일기 페이지 2장(`1_7.1` 1·2면)을 한 페이지의 섹션 10개로 통합.

## 가이드 영상

일기 활동 페이지에는 **활동 진행을 안내하는 가이드 영상**이 함께 제공됩니다. 데이터 모델의 `Page.guideVideoUrl` 필드로 정의되며, 본문 상단에 `GuideVideo` 컴포넌트가 토글 가능한 YouTube 임베드로 표시됩니다. (원본 종이 콘텐츠의 우측 상단 QR 코드와 동일한 역할)

---

## 사용자 흐름

1. 홈에서 Day 7 → "오늘 하루 어땠나요?" 클릭
2. 상단에 가이드 영상이 펼쳐진 상태로 표시 (펼침/접힘 토글 가능)
3. 안내 문구 + 10개 섹션이 카드 형태로 표시
3. 섹션 유형에 따라 다른 입력 UI:
   - **radio**: 큰 버튼(택 1) — 선택 시 핑크색으로 강조
   - **checkbox**: 큰 버튼(다중 선택) — 선택 시 ✓ 마크 + 핑크색
   - **text**: 큰 input 필드
   - **textarea**: 다중 행 입력 (기본 4행, `rows` 옵션으로 조정)
4. 입력값 변경 시 400ms 디바운스로 자동 저장 (`inputStore`)
5. 페이지 재진입 시 입력값 자동 복원
6. 하단 **"일기 작성 완료"** 버튼으로 진행 상태 저장 후 홈 이동

## 핵심 구현 사항

### 답변 데이터 구조
```ts
type DiaryAnswers = Record<string, string | string[]>;
// radio/text/textarea → string
// checkbox → string[]
```

`section.key`를 키로 사용 (예: `mood`, `breakfast`, `exercise`).

### 섹션별 분기 렌더링
`section.kind` discriminated union을 사용하여 UI 분기:
- `radio` / `checkbox`: 큰 버튼 (min-h-touch = 48px)
- `text`: 단일 행 input
- `textarea`: 다중 행 textarea (`resize-y`로 사용자가 높이 조절 가능)

모든 입력은 `text-base`(20px) 적용 — 어르신 가독성.

### 디바운스 자동 저장
```ts
useEffect(() => {
  if (Object.keys(answers).length === 0) return;
  const t = setTimeout(() => { void inputStore.set(pageNo, answers); }, 400);
  return () => clearTimeout(t);
}, [pageNo, answers]);
```

- 빠른 타이핑 중에는 저장 미발생 → 400ms 멈춤 후 저장
- 초기 마운트 시 빈 객체 저장 방지 (`Object.keys.length === 0` 가드)

### 초기 복원
마운트 시 `inputStore.get(pageNo)`로 저장된 답변을 불러와 상태로 설정.

## 알려진 제약 / 후속 작업

1. **음성 입력(STT) 없음** — 어르신이 키보드 입력을 어려워할 경우를 대비한 음성 입력 미구현. 사용성 테스트 후 우선순위 결정.
2. **이미지 첨부 없음** — 종이 일기 시절엔 사진을 직접 붙일 수 있었으나, 디지털 버전엔 이미지 업로드 미구현.
3. **날짜 자동 기록 없음** — 일기에 작성일이 자동으로 기록되지 않음. 향후 `completedAt` 메타데이터를 progress에 함께 저장하는 방향 고려.
4. **이전 일기 다시 보기 없음** — 작성한 일기를 모아 보는 화면 미구현. 어르신이 본인의 변화를 회고하기 좋은 기능이므로 후속 우선순위 높음.
5. **공유·인쇄 기능 없음** — 가족과 공유하거나 인쇄해서 보관하는 기능 미구현.
6. **자유 일기 페이지(`1_7.2` 이번 주도 잘 지내셨습니까?) 미구현** — 점선 노트 위에 손글씨로 쓰는 자유 일기 페이지는 PoC 범위에서 제외. 향후 색칠 캔버스를 응용한 손글씨 입력 또는 단순 textarea로 추가 가능.
