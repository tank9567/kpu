# 22. 빈칸 활동 컴포넌트 (BlankActivity)

> 4종 활동 중 두 번째 — 속담·문장의 빈칸을 채우고 정답을 확인하는 활동.

## 개요

| 항목 | 값 |
|------|-----|
| 활동 타입 | `blank` |
| 대표 페이지 | `1_2.1 방귀쟁이 며느리 시집가는 날 1` |
| 메인 인터랙션 | 빈칸 텍스트 입력 + 정답 확인 |
| 자동 저장 | localStorage |

---

## 활동 데이터 스키마

```ts
type BlankActivity = {
  type: 'blank';
  prompt: string;          // 안내 문구
  image?: string;          // 선택: 삽화 URL
  questions: Array<{
    template: string;      // 빈칸을 "{}"로 표시한 문장
    answers: string[];     // 각 빈칸의 정답 (template의 {} 갯수와 일치)
  }>;
};
```

샘플 (`app/src/data/issues/5-4.json` Day 2):
```json
{
  "type": "blank",
  "prompt": "속담을 읽어 보고, 빈칸에 들어갈 단어를 적어보세요.",
  "questions": [
    { "template": "벙어리 {} 년, 귀머거리 {} 년, 장님 {} 년", "answers": ["3", "3", "3"] },
    { "template": "때리는 {} 보다 말리는 {} 가 더 밉다.", "answers": ["시어머니", "시아버지"] },
    { "template": "시집살이는 {} 도 안 한다.", "answers": ["개"] }
  ]
}
```

## 가이드 영상

빈칸 활동 페이지에는 **활동 진행을 안내하는 가이드 영상**이 함께 제공됩니다. 데이터 모델의 `Page.guideVideoUrl` 필드로 정의되며, 본문 상단에 `GuideVideo` 컴포넌트가 토글 가능한 YouTube 임베드로 표시됩니다. (원본 종이 콘텐츠의 우측 상단 QR 코드와 동일한 역할)

```ts
type Page = {
  pageNo: string;
  category: string;
  title: string;
  guideVideoUrl?: string;   // 활동 가이드 영상 (선택)
  activity: Activity;
};
```

## 사용자 흐름

1. 홈에서 Day 2 → "방귀쟁이 며느리 시집가는 날" 클릭
2. 상단에 가이드 영상이 펼쳐진 상태로 표시 (펼침/접힘 토글 가능)
3. 안내 문구 + 빈칸이 포함된 속담들이 카드 목록으로 표시
3. 빈칸 input에 답을 입력하면 자동으로 localStorage에 저장
4. 모든 빈칸 입력 시 **"정답 확인"** 버튼 활성화
5. 클릭하면:
   - 정답인 빈칸: 초록 테두리 + ✅
   - 오답인 빈칸: 빨강 테두리 + ❌
   - 카드 하단에 정답 노출
6. **"다시 풀어보기"** 버튼으로 정답 표시 해제 (입력값은 유지)
7. 하단의 완료 버튼으로 진행 상태 저장

## 핵심 구현 사항

### 빈칸 분리 로직
`template.split('{}')`로 텍스트 조각을 분리하고, 사이에 `BlankInput`을 끼워 넣는 방식:

```tsx
// "벙어리 {} 년, 귀머거리 {} 년, 장님 {} 년"
parts = ['벙어리 ', ' 년, 귀머거리 ', ' 년, 장님 ', ' 년']
// parts.length - 1 = 3 (= 빈칸 개수)
```

각 질문의 `answers` 배열 길이와 빈칸 개수가 일치해야 함 (콘텐츠 작성자 책임).

### 자동 저장
입력값 변경 시마다 `inputStore.set(pageNo, answers)` 호출 — `localStorage`의 `input:5-4:1_2.1`에 JSON 문자열로 저장. 페이지 재진입 시 자동 복원.

### 정답 비교
`value.trim() === correct.trim()`으로 단순 비교. 공백 차이는 무시.

### 정답 확인 버튼 활성화 조건
모든 빈칸이 비어있지 않을 때만 활성화 (`allAnswered` flag).

## 알려진 제약 / 후속 작업

1. **동의어·유의어 처리 안 함** — 예: "시어머니" / "시母" 같은 변형은 모두 오답 처리. 향후 `answers`를 배열의 배열(`string[][]`)로 확장하면 다중 정답 지원 가능.
2. **부분 정답 점수화 없음** — 한 문제당 모든 빈칸이 맞아야 정답으로 카운트되는 식의 점수화는 미구현. 단순히 빈칸별 ✅/❌만 표시.
3. **힌트 기능 없음** — 어르신이 막혔을 때 도움을 받을 수 있는 힌트 UI 없음.
4. **이미지 삽화 미사용** — 스키마에 `image?` 필드는 있으나 현재 콘텐츠엔 없음. 필요 시 안내 문구 아래에 큰 이미지로 표시됨.
