# 23. 색칠 활동 컴포넌트 (ColoringActivity)

> 4종 활동 중 세 번째 — 만다라 도안의 **영역을 클릭(터치)** 하면 선택된 색으로 칠해지는 방식.

## 개요

| 항목 | 값 |
|------|-----|
| 활동 타입 | `coloring` |
| 대표 페이지 | `1_3.1 만다라 보면서 마음 명상하기` |
| 메인 인터랙션 | 인라인 SVG 영역 클릭 → 해당 영역만 색 채움 |
| 구현 방식 | 영역 단위 SVG path + React state |
| 저장 | localStorage (영역 → 색상 매핑 JSON) |

> **설계 결정**: 초기엔 `react-konva` 기반 자유 그리기로 만들었으나, 어르신이 손가락으로 정밀하게 색칠하는 부담이 커서 **영역 클릭 방식으로 전면 전환**. 가벼운 입력으로 만족스러운 결과가 나오도록.

---

## 가이드 영상

색칠 활동 페이지에는 **활동 진행을 안내하는 가이드 영상**이 함께 제공됩니다. 데이터 모델의 `Page.guideVideoUrl` 필드로 정의되며, 본문 상단에 `GuideVideo` 컴포넌트가 토글 가능한 YouTube 임베드로 표시됩니다. (원본 종이 콘텐츠의 우측 상단 QR 코드와 동일한 역할)

## 활동 데이터 스키마

```ts
type ColoringActivity = {
  type: 'coloring';
  templateImage: string;     // (현재 PoC에선 내장 Mandala 컴포넌트가 항상 사용됨, 향후 확장 시 도안 선택자로 활용)
  exampleImage?: string;     // 선택: 색칠 예시 이미지
  palette?: string[];        // 선택: 기본 팔레트 hex 코드 배열
};
```

샘플 (`app/src/data/issues/5-4.json` Day 3):
```json
{
  "type": "coloring",
  "templateImage": "/assets/mandala-1.svg",
  "palette": ["#E94B6B", "#F4A261", "#E9C46A", "#2A9D8F", "#264653", "#9B5DE5"]
}
```

## 사용자 흐름

1. 홈에서 Day 3 → "만다라 보면서 마음 명상하기" 클릭
2. 상단에 가이드 영상이 펼쳐진 상태로 표시
3. 안내 문구 아래에 만다라 SVG (360 × 360 px)
4. 팔레트에서 색상 선택 (마지막 칸 ↺ = 흰색, 지우개 역할)
5. 만다라의 한 영역을 **살짝 터치/클릭** → 선택된 색으로 채워짐
6. 다른 영역도 차례로 클릭하여 색칠
7. 이미 칠해진 영역을 다시 클릭하면 새 색으로 덮어쓰기
8. **되돌리기** — 마지막 클릭 이전 상태로 복원 (이력 무제한)
9. **전부 지우기** — 모든 영역을 다시 흰색으로
10. 모든 변화는 localStorage에 즉시 자동 저장 → 재진입 시 복원
11. 하단 완료 버튼으로 진행 상태 저장

---

## 만다라 영역 구성 (11개)

| 영역 ID | 설명 | 모양 |
|---------|------|------|
| `outer` | 외곽 큰 원 (r=180) | 가장 바깥 배경 |
| `mid` | 중간 원 (r=140) | 꽃잎 사이 배경 |
| `petal-0`~`petal-7` | 8각 회전 대칭 꽃잎 | 외곽~중심 사이 |
| `center` | 중심 원 (r=25) | 가운데 |

SVG z-order: `outer` → `mid` → `petals` → `center` 순으로 위에 쌓여, 클릭 시 가장 위 영역이 선택됨.

## 핵심 구현 사항

### 데이터 모델
```ts
type Fills = Record<string, string>;  // 예: { 'petal-0': '#E94B6B', 'center': '#F4A261' }
```

- 영역 ID를 키로, 색상 hex를 값으로 저장
- 색칠 안 한 영역은 키 없음 (기본 `#ffffff`)
- 흰색으로 다시 칠하면 키는 남고 값이 `#ffffff`로 갱신 (지우개 효과)

### 영역 클릭 처리
```ts
const onRegionClick = (region: string) => {
  setHistory((h) => [...h, fills]);    // 직전 상태를 이력에 push
  setFills((f) => ({ ...f, [region]: color }));
};
```

### 되돌리기 (Undo)
- 클릭마다 직전 `fills` 스냅샷을 `history` 배열에 push
- `undo()` 호출 시 마지막 스냅샷을 복원하고 pop
- 메모리 사용: 영역이 11개뿐이므로 무제한 이력 유지해도 부담 없음

### 자동 저장
- `fills` 변경 시마다 `inputStore.set(pageNo, fills)` (localStorage)
- 초기 마운트 시 빈 `{}`를 덮어쓰지 않도록 `loaded` 플래그로 가드
- 디바운스 없이 즉시 저장 (Object 크기가 작음)

### 터치 처리
SVG path/circle 요소에 `onClick`과 `onTouchEnd` 둘 다 바인딩. 모바일에서 클릭 지연(300ms) 회피 및 일관된 동작 보장.

## 검증 방법

- [ ] 홈 → Day 3 진입 시 만다라가 흰색 도안으로 보임 (선이 분명히 표시)
- [ ] 팔레트에서 색상 선택 시 굵은 테두리로 표시
- [ ] 도안의 한 영역(꽃잎, 중심, 외곽 등) 클릭 시 즉시 그 영역만 채워짐
- [ ] 손가락 터치로도 동일 동작 (모바일)
- [ ] 다른 영역 클릭 시 그 영역만 추가로 채워짐 (이전 색칠은 유지)
- [ ] 이미 칠해진 영역을 다른 색으로 다시 클릭하면 덮어쓰기
- [ ] 흰색(↺) 선택 후 영역 클릭하면 지워짐
- [ ] 되돌리기 1회 = 마지막 클릭 1회 취소
- [ ] 전부 지우기 = 모든 영역 흰색으로 초기화
- [ ] 페이지 새로고침 후 재진입 시 색칠 결과 복원 (localStorage)
- [ ] 완료 버튼으로 진행 상태 저장 및 홈 이동

## 알려진 제약 / 후속 작업

1. **만다라 도안 1종 고정** — `templateImage` 필드가 있긴 하나 현재는 내장 `<Mandala>` 컴포넌트가 항상 렌더링됨. 도안이 늘어나면 컴포넌트 레지스트리(`{ 'mandala-1': MandalaA, 'mandala-2': MandalaB }`) 또는 SVG 정적 import 매핑 필요.
2. **자유 그리기 모드 없음** — 영역 클릭 방식만 지원. 손글씨로 그림을 그리고 싶은 사용자에겐 별도 활동 유형(예: `freedraw`)을 신설하는 방향 권장.
3. **그라데이션·텍스처 없음** — 단색 fill만 지원. PoC 단계로 충분.
4. **결과물 내보내기(이미지 저장) 없음** — `<svg>`를 PNG로 변환하여 다운로드/공유하는 기능 미구현.
5. **영역 강조 미리보기 없음** — 호버 시 영역 외곽선이 강조되는 등의 시각 피드백은 `hover:opacity-90`으로만 처리. 터치 환경에선 호버 효과가 없으므로 어르신에겐 큰 차이 없음.
6. **`templateImage` 자산 파일은 더 이상 직접 사용되지 않음** — `app/src/assets/mandala-1.svg`는 이전 자유 그리기 방식의 잔재. 향후 정리 또는 미리보기 썸네일 용도로 재활용 검토.
